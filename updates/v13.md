# v13 Migration — Self-Evolving Scaffolding

One-time migration for existing Antigravity projects. Run this when the bootstrapper detects a version < 13.

## Pre-flight

```bash
git clone --depth 1 https://github.com/ButchMenzies/antigravity-project-setup.git /tmp/ag-setup
```

---

## Step 1: Detect Project Type

```bash
ls package.json pyproject.toml requirements.txt go.mod Cargo.toml Gemfile 2>/dev/null
```

- **If any manifest found** → **Code project**
- **If only markdown/docs** → Check AGENT.md:
  ```bash
  grep -l "Audience" .agent/AGENT.md 2>/dev/null
  ```
  If found → **Workspace project**. Otherwise → ask the user.

Note the project type — it determines which template and which workflows/skills to install.

---

## Step 2: Overwrite All Workflows

```bash
cp /tmp/ag-setup/.agent/workflows/*.md .agent/workflows/
```

Then remove type-inappropriate workflows:

**Workspace projects:**
```bash
rm -f .agent/workflows/test.md
rm -f .agent/workflows/security-review.md
rm -f .agent/workflows/ux-design.md
```

**Code projects** — no removals needed (keep all).

Remove obsolete workflows (all project types):
```bash
rm -f .agent/workflows/update.md
rm -f .agent/workflows/workspace-audit.md
```

---

## Step 3: Overwrite All Skills

**Code projects** — install all skills:
```bash
for skill in /tmp/ag-setup/skills/*/; do
  name=$(basename "$skill")
  case "$name" in bundles) continue;; esac
  mkdir -p ".agent/skills/$name"
  cp -r "$skill"* ".agent/skills/$name/"
done
```

**Workspace projects** — install selected skills:
```bash
for skill in planning copywriting ux-design voice-notes-triage offer-strategy lead-strategy write-code code-review architecture-change; do
  if [ -d "/tmp/ag-setup/skills/$skill" ]; then
    mkdir -p ".agent/skills/$skill"
    cp -r "/tmp/ag-setup/skills/$skill/"* ".agent/skills/$skill/"
  fi
done
```

---

## Step 4: Replace AGENT.md

This is the key step. Instead of patching the old AGENT.md, replace it with the v13 template and re-inject project-specific content.

### 4a. Extract project-specific content

Read the current `.agent/AGENT.md`. Look for and save the content from these sections:

- `## Your Role` — save the paragraphs (if present and not the default template text)
- `## Project Principles` — save all content
- `### Terminology` — save the table rows (if any real entries exist)
- `## What This Project Is` / `## Project Overview` — save everything
- `## Tech Stack` / `## Architecture` / `## Stack` — save everything
- `## Don't Be Lazy` — check for any custom rules beyond the template defaults; save those

If a section is empty, uses placeholder text, or doesn't exist — mark it as "needs auto-fill."

### 4b. Write the new AGENT.md

Copy the appropriate template:
```bash
cp /tmp/ag-setup/templates/AGENT-code.md .agent/AGENT.md
# or: cp /tmp/ag-setup/templates/AGENT-workspace.md .agent/AGENT.md
```

### 4c. Inject extracted content

For each section that had real content in 4a, replace the placeholder in the new AGENT.md with the saved content.

### 4d. Auto-fill empty sections

For sections marked "needs auto-fill" in 4a:

- **Your Role** → leave the template default (it's good enough)
- **Project Principles** → leave the placeholder comment
- **Terminology** → leave the placeholder comment
- **What This Project Is** → scan the project:
  ```bash
  ls -la
  cat package.json 2>/dev/null | head -5
  cat README.md 2>/dev/null | head -20
  ```
  Write a brief description and folder structure based on what you find.
- **Tech Stack** → read manifest files (`package.json`, `requirements.txt`, etc.) and list the key technologies
